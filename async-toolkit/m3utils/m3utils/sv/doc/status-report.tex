\begin{abstract}
This report summarizes the development of the SystemVerilog front-end
tool suite (\texttt{svfe}) within the \texttt{m3utils} repository.
Work began on March 5, 2026 and is ongoing.  In three days,
the project has produced a complete SystemVerilog parser,
a standalone preprocessor, an RTL lint tool, a BDD-based logic
synthesizer with formal equivalence checking, a SOP equation
generator, a hand-written MOS~6502 CPU model verified against the
Klaus Dormann functional test suite, and a cycle-accurate 6502
emulator with BDD-generated C evaluation backends.
\end{abstract}

\section{Project Overview}

The \texttt{sv/} subtree of \texttt{m3utils} implements a suite of
tools for parsing, analyzing, synthesizing, and verifying
SystemVerilog RTL designs.  All tools share a common parser front-end
(\texttt{svfe}) built with the Caltech \texttt{parserlib}
infrastructure in Modula-3.  Analysis and synthesis passes are
implemented in Scheme, running on the \texttt{mscheme} interpreter
embedded in the \texttt{svsynth} program, which extends mscheme with
BDD primitives from the CM3 BDD library.

\subsection{Timeline}

\begin{description}[leftmargin=3cm,style=nextline]
\item[March 5] Parser front-end (Phase~1 and~2), preprocessor,
  Scheme analysis libraries (base, lint, codegen), BDD synthesis
  engine, gate-level emitter, round-trip verification, ibex test
  suite (30/30 RTL, 161/161 primitives).
\item[March 6] Bit-level BDD synthesis with LRM-correct widths,
  svlint (7 rules), sveqc (equivalence checker), 6502 ALU synthesis,
  BDD-to-C code generation, 6502 emulator, carry-cut multi-level
  synthesis, MUX/decoder decomposition, SOP output via Caltech
  espresso library, syntax-directed BDD decomposition, Modula-3
  preprocessor rewrite.
\item[March 7] 6502 CPU RTL debugging---8 bugs found and fixed via
  iverilog simulation.  Klaus Dormann 6502 functional test passes
  at 102 million cycles.
\end{description}

\subsection{BDD Synthesis Engine (\texttt{svsynth})}

\begin{tabular}{@{}ll@{}}
\toprule
Source & \texttt{sv/svsynth/src/} \\
Lines  & $\sim$491 (Modula-3, BDD primitive bindings) \\
Base   & CM3 BDD library (\texttt{m3-libs/BDD/}) \\
\bottomrule
\end{tabular}

\medskip\noindent
Extends mscheme with: \texttt{bdd-var}, \texttt{bdd-and},
\texttt{bdd-or}, \texttt{bdd-not}, \texttt{bdd-ite},
\texttt{bdd-equal?}, \texttt{bdd-hash}, \texttt{bdd-id},
\texttt{bdd-size}, \texttt{bdd-restrict}, etc.

Key synthesis features:
\begin{itemize}[nosep]
\item Bit-level BDD construction for multi-bit signals (up to 32 bits)
\item Carry-cut mechanism for multi-level synthesis of wide arithmetic
\item Decoder + MUX case decomposition for large \texttt{case} statements
\item Syntax-directed BDD decomposition to prevent blowup on wide equality
\item Shannon expansion for gate-level output
\end{itemize}

\subsection{RTL Lint Tool (\texttt{svlint})}

Seven lint rules, tested on 7 purpose-built test cases and validated
on all 30 ibex RTL files:

\begin{enumerate}[nosep]
\item Undriven output ports
\item Unused declared signals
\item Multiple drivers on same signal
\item Latch inference (incomplete \texttt{if}/\texttt{case} in combinational blocks)
\item Width mismatches in assignments
\item Blocking assignments in \texttt{always\_ff}
\item Combinational loops (cycle detection in assign dependency graph)
\end{enumerate}

\subsection{Equivalence Checker (\texttt{sveqc})}

Self-equivalence and two-design formal equivalence checking via BDD
comparison.  Flow: parse $\to$ build BDDs $\to$ emit gate-level SV
$\to$ re-parse $\to$ build BDDs $\to$ compare bit-by-bit with
\texttt{bdd-equal?}.

\begin{tabular}{@{}ll@{}}

Style  & Multi-cycle FSM, single-file, no submodules \\
ISA    & Full NMOS 6502 including BCD decimal mode \\
\bottomrule

\medskip\noindent
The CPU implements:
\begin{itemize}[nosep]
\item All 56 official opcodes (151 opcode bytes) across 13 addressing modes
\item BCD arithmetic for ADC and SBC with correct NMOS flag behavior
\item IRQ, NMI (edge-triggered), and BRK interrupt handling
\item Stack operations (PHA/PLA/PHP/PLP, JSR/RTS, RTI)
\item All read-modify-write instructions (ASL, ROL, LSR, ROR, INC, DEC)
\item Page-crossing address calculation with correct carry propagation
\end{itemize}

\subsection{Verification}

\begin{tabular}{@{}lll@{}}
\toprule
Test & Result & Notes \\
\midrule
Simple test (5 tests) & \textcolor{pass}{\textbf{PASS}} & LDA, STA, ADC, INX, JSR/RTS \\
Klaus Dormann functional & \textcolor{pass}{\textbf{PASS}} & 102,236,223 cycles, all opcodes + BCD \\
\bottomrule
\end{tabular}

\medskip\noindent
Eight bugs were found and fixed during simulation:
\begin{enumerate}[nosep]
\item JSR return address (push PC$-$1, not PC)
\item BRK push order (PCH before PCL)
\item RTI stack pull sequence
\item S\_DECODE to S\_EXEC transition for implied-mode instructions
\item BRK/IRQ handler: split S\_BRK1 into S\_BRK1+S\_BRK2 to fix
      double non-blocking assignment clobbering stack write address
\item Page-crossing carry propagation in abs,X / abs,Y / (ind),Y
      (use 16-bit addition instead of concatenation)
\item Accumulator-mode shifts reading stale \texttt{exec\_operand}
      instead of \texttt{A} directly
\item Missing BCD decimal mode for ADC and SBC
\end{enumerate}

\section{Architecture}

The tool flow is:

\begin{enumerate}[nosep]
\item \textbf{Preprocessing:} \texttt{svpp} handles \texttt{`define},
  \texttt{`ifdef}, \texttt{`include}, token pasting.
\item \textbf{Parsing:} \texttt{svfe --scm} produces an S-expression AST.
\item \textbf{Analysis:} Scheme libraries walk the AST for lint checks,
  signal extraction, width inference.
\item \textbf{Synthesis:} \texttt{svbv.scm} builds per-bit BDDs for
  each output signal.  Carry cuts, MUX decomposition, and
  syntax-directed decomposition keep BDD size manageable.
\item \textbf{Emission:} BDDs are converted to gate-level SV
  (\texttt{svemit.scm}), C code (\texttt{svemit-c.scm}), or SOP
  equations (\texttt{svsop.scm}).
\item \textbf{Verification:} Round-trip BDD comparison confirms
  functional equivalence between behavioral and synthesized designs.
\end{enumerate}

%======================================================================
\section{Known Limitations}

\begin{itemize}[nosep]
\item Parser does not yet support \texttt{import "DPI-C"} (with prototype)
  or packed unions.
\item BDD synthesis is limited to combinational cones; sequential
  equivalence checking is not implemented.
\item BDD blowup on designs with $>$24 independent inputs
  (mitigated by carry cuts and decomposition, but not eliminated).
\item The 6502 CPU model does not implement unofficial/illegal opcodes.
\item No Verilator integration yet (would speed simulation significantly).
\end{itemize}
